# 动态配置功能示例

[English Version](examples/dynamic_config/README.md)

## 问题背景

在使用 zlog 时，传统方式要求在配置文件中提前定义所有的日志分类和过滤规则。这对于以下场景不够灵活：

- **插件化架构**：插件在运行时加载，无法提前在配置文件中定义
- **模块解耦**：每个模块希望独立管理自己的日志配置
- **动态调整**：需要根据运行时条件（如调试模式、用户权限等）动态改变日志级别

## 解决方案

zlog 提供了 **从字符串加载配置** 的 API，允许在运行时动态构建和重载配置：

```c
/* 从配置字符串初始化 zlog */
int zlog_init_from_string(const char *config_string);

/* 从配置字符串重新加载配置 */
int zlog_reload_from_string(const char *config_string);
```

## 快速示例

```c
#include "zlog.h"

int main() {
    /* 步骤1：使用最小化配置初始化 */
    const char *config =
        "[global]\n"
        "strict init = false\n"
        "\n"
        "[formats]\n"
        "simple = \"%d [%c] %m%n\"\n"
        "\n"
        "[rules]\n"
        "*.INFO >stdout; simple\n";

    zlog_init_from_string(config);

    /* 步骤2：运行时动态添加新模块 */
    const char *new_config =
        "[global]\n"
        "strict init = false\n"
        "\n"
        "[formats]\n"
        "simple = \"%d [%c] %m%n\"\n"
        "detailed = \"%d [%V] [%c] [%f:%L] %m%n\"\n"
        "\n"
        "[rules]\n"
        "module_a.INFO >stdout; simple\n"
        "module_b.DEBUG \"./logs/module_b.log\"; detailed\n"
        "module_b.WARN >stdout; simple\n";

    zlog_reload_from_string(new_config);

    /* 步骤3：使用新添加的分类 */
    zlog_category_t *cat = zlog_get_category("module_b");
    zlog_info(cat, "模块 B 已加载");

    zlog_fini();
    return 0;
}
```

## 使用场景示例

### 场景1：插件系统

```c
void load_plugin(const char *plugin_name) {
    char config[4096];

    /* 构建包含新插件的完整配置 */
    build_full_config_with_plugin(config, sizeof(config), plugin_name);

    /* 重新加载配置，新插件的日志规则立即生效 */
    zlog_reload_from_string(config);

    /* 插件可以立即使用自己的日志分类 */
    zlog_category_t *cat = zlog_get_category(plugin_name);
    zlog_info(cat, "Plugin %s loaded", plugin_name);
}
```

### 场景2：调试模式切换

```c
void toggle_debug_mode(bool enable) {
    char config[4096];

    if (enable) {
        /* 所有模块切换到 DEBUG 级别 */
        snprintf(config, sizeof(config),
            "[global]\n"
            "strict init = false\n"
            "[formats]\n"
            "debug = \"%%d [DEBUG] [%%c] %%m%%n\"\n"
            "[rules]\n"
            "*.DEBUG >stdout; debug\n"
            "*.DEBUG \"./logs/debug.log\"; debug\n");
    } else {
        /* 恢复正常级别 */
        snprintf(config, sizeof(config),
            "[global]\n"
            "strict init = false\n"
            "[formats]\n"
            "normal = \"%%d [%%V] [%%c] %%m%%n\"\n"
            "[rules]\n"
            "*.INFO >stdout; normal\n");
    }

    zlog_reload_from_string(config);
}
```

### 场景3：根据用户权限调整日志详细程度

```c
void set_log_level_by_user_role(const char *role) {
    char config[4096];

    if (strcmp(role, "admin") == 0) {
        /* 管理员看到所有详细日志 */
        snprintf(config, sizeof(config),
            "[global]\n"
            "strict init = false\n"
            "[formats]\n"
            "admin_fmt = \"%%d [%%V] [%%c] [%%f:%%L %%U()] %%m%%n\"\n"
            "[rules]\n"
            "*.DEBUG >stdout; admin_fmt\n"
            "*.DEBUG \"./logs/admin.log\"; admin_fmt\n");
    } else {
        /* 普通用户只看重要信息 */
        snprintf(config, sizeof(config),
            "[global]\n"
            "strict init = false\n"
            "[formats]\n"
            "user_fmt = \"%%d [%%V] %%m%%n\"\n"
            "[rules]\n"
            "*.INFO >stdout; user_fmt\n");
    }

    zlog_reload_from_string(config);
}
```

## 关键要点

1. **配置格式**：配置字符串的格式与配置文件完全相同
2. **完整替换**：每次 `reload_from_string` 都会完全替换当前配置，不是增量添加
3. **重新获取分类**：reload 后建议重新调用 `zlog_get_category()` 获取分类句柄
4. **线程安全**：`zlog_reload_from_string()` 是线程安全的，但建议在主线程调用
5. **百分号转义**：在 C 字符串中，格式化符号的 `%` 需要写成 `%%`

## 优势

与静态配置文件相比，动态配置的优势：

| 特性 | 静态配置文件 | 动态配置字符串 |
|------|------------|----------------|
| 灵活性 | 需要提前定义所有分类 | 运行时根据需要添加 |
| 解耦性 | 所有模块耦合在一个文件 | 每个模块独立管理 |
| 调试便利性 | 需要修改文件并 reload | 可通过命令即时调整 |
| 适用场景 | 配置相对固定的系统 | 插件化、动态模块化系统 |

## 常见问题

**Q: 可以只添加新分类而不影响现有分类吗？**
A: 不行，每次 reload 都是完整替换。需要在新配置中包含所有需要保留的规则。

**Q: reload 会丢失正在写入的日志吗？**
A: 不会，reload 期间会短暂阻塞，但不会丢失日志。

**Q: 配置字符串可以从网络接收吗？**
A: 可以，但请注意验证配置内容的安全性。
